Chapter 13: CSS Cascading

Rule 1: The Last Declaration Wins

If two selectors have the same specificity, the one defined *last* in the CSS file is applied. 

/* 1. First we set it to Blue */
.last-win-example p {
  color: blue;
}

/* 2. Then we set it to Red. Since this is last, the text is Red. */
.last-win-example p {
  color: red;
}

This paragraph was first colored blue, then red. It appears red.

Rule 2: Specificity (Weight)

If selectors are different, the "more specific" one wins.
ID (#) > Class (.) > Tag (p) 

/* Tag Selector (Weight: 1) */
.specificity-example p {
  color: black;
}

/* Class Selector (Weight: 10) -> Wins over Tag */
.specificity-example .highlight {
  color: red;
}

/* ID Selector (Weight: 100) -> Wins over Class */
#specific-id {
  color: green;
}

I have a Class and a Tag selector. Class wins (Red).

I have an ID, a Class, and a Tag selector. ID wins (Green).

Rule 3: Inline Styles

Styles written directly in the HTML `style=""` attribute override external and internal CSS. 

/* External CSS tries to make it blue */
.inline-example p {
  color: blue;
}

/* But HTML has style="color: red" */

This paragraph is red because of the inline style, ignoring the CSS.

Rule 4: The !important Declaration

`!important` overrides all specificity rules. Use strictly for overriding styles you cannot otherwise control. 

/* ID usually wins (Blue) */
#unique-id {
  color: blue;
}

/* But !important forces this Class to win (Red) */
.important-example .special {
  color: red !important;
}

I have an ID (Blue), but the Class has !important (Red). So I am Red.

Inheritance

Some properties (like text color and font) are inherited by child elements from their parents. 

/* We only style the parent. Children inherit these values. */
.parent {
  color: blue;
  font-family: sans-serif;
}
I am the Parent (Blue).
I am the Child. I inherited the Blue color from my parent.
I am the Grandchild. I also inherited the Blue color.
